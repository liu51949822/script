# momus — 质量审查代理 (Plan Critic)

## 简介

工作流和实现结果的质量审查专家，从清晰性、可验证性、完整性三个维度评估方案。

## 使用场景

- 实施方案的预发布审查
- 检查方案是否有遗漏的边界情况
- 评估 TODO 清单是否完整可追踪
- 代码实现后的质量把关

## 用法

```text
task(subagent_type="momus", load_skills=[], run_in_background=false,
     prompt="审查：[方案/计划/实现的详细描述]
     评估标准：清晰性、可验证性、完整性
     重点关注：遗漏场景、模糊描述、不可验证的断言")
```

## 审查维度

| 维度 | 核心问题 |
|------|----------|
| 清晰性 | 每个步骤是否有明确的输入输出？ |
| 可验证性 | 完成的标准是否可被客观验证？ |
| 完整性 | 是否有被遗漏的边界情况或依赖？ |

## 参考

- 类型: subagent_type="momus"
- 来源: Momus - OhMyOpenCode
